Where Are the Neurons in Llama? — MLP vs Transformer

You understand 128 neurons × 784 weights = 100,480 params. The exact same formula applies to Llama 7B's 11,008 FFN neurons × 4,096 weights. The numbers are bigger — the math is identical.

🔑 Answer: Llama's Feed-Forward Network (FFN) IS an MLP!

Every transformer block has an FFN sub-layer that is literally a 2-layer MLP: MLP: input(784) → W₁(784×128) → ReLU → W₂(128×64) → ReLU → W₃(64×10) → output(10) Llama FFN: input(4096) → gate_proj(4096×11008) → SiLU → × up_proj(4096×11008) → down_proj(11008×4096) → output(4096) SAME PATTERN! Just bigger numbers, SwiGLU instead of ReLU, and repeated 32 times.

Every one of these 7 matrices is a layer of neurons. Each row = one neuron. Each neuron = weighted sum of ALL its inputs. Q, K, V, O, gate, up, down — ALL follow neurons × inputs.

🤔 Why Does Llama Keep 4096 Everywhere? — "The Residual Stream"

🔵 MLP — The "Funnel" (Shrinking)

784 → 128 → 64 → 10
SHRINKS at every layer
Why? The goal is to COMPRESS information down to one answer (a single digit). Each layer throws away irrelevant details. By the output layer, you just have 10 numbers — "which digit?"
Like a detective narrowing suspects:
784 clues → 128 patterns → 64 theories → 1 answer

🟣 Llama — The "Residual Stream" (Constant Width)

4096 → 4096 → 4096 → ... (×32)
STAYS THE SAME throughout
Why? The goal is to REFINE information, not compress it. Llama needs to predict the NEXT token at every position — it needs FULL information at all times. You can't predict the next word from a 10-number summary!
Like editing an essay 32 times:
4096-word draft → refined → refined → ... → final draft
Same length throughout. Just better each pass.
// THE KEY INSIGHT: The FFN expands INTERNALLY, then compresses back Each Llama block does: 4096 ──→ FFN expands to 11,008 ──→ FFN compresses back to 4096 The 11,008-neuron hidden layer IS a funnel inside each block! 4096→11008→4096. But the RESIDUAL CONNECTION (x + block(x)) requires same size in and out. So the RESIDUAL STREAM stays at 4096, but the FFN internally expands to 11,008. Analogy: Think of the 4096 as a highway — constant width, carries information across all 32 blocks. Each FFN is an off-ramp — expands to a WIDER processing area (11,008), does intensive computation, then merges back onto the highway at the SAME width (4096). MLP: Highway starts wide (784), gets narrow (128, 64), ends at exit (10). Llama: Highway stays at constant width (4096) for 32 exits, off-ramps expand/contract, merge back.
🔵

MLP: Compress to Answer

Each layer has FEWER neurons. Information is LOST at every step — but that's OK because only the final answer matters. 784→128→64→10.

🟣

Llama: Refine at Full Resolution

Every layer keeps 4096 neurons. The residual connection ADDS the block's output to the input. Information is never lost — only refined. 4096→4096→4096.

💡

The FFN IS a mini-funnel!

Inside each block, the FFN expands 4096→11008 then compresses 11008→4096. The expansion gives it "thinking space." The compression brings it back to highway width.

🧮 THE UNIVERSAL FORMULA — Same Math, Different Scale

For ANY linear layer in ANY neural network: params = output_neurons × (input_features + has_bias) MLP Hidden 1: 128 neurons × (784 inputs + 1 bias) = 100,480 Llama gate_proj: 11,008 neurons × (4,096 inputs + 0 bias) = 45,088,768 SAME FORMULA. 450× larger numbers. 32× more layers. That's the difference.
🔵

In the MLP...

"Neurons" are explicit: 128 neurons in Hidden Layer 1. Each has 784 weights (one per pixel). You can count them: 128 × 785 = 100,480.

🟣

In Llama...

"Neurons" are the OUTPUT DIMENSION of each linear layer. gate_proj[4096→11008]: 11,008 neurons, 4,096 weights each. Same concept, different name.

💡

Why the different language?

ML papers say "hidden size" instead of "neurons" because at transformer scale, you think in matrix dimensions, not individual neurons. But the math never changed.
